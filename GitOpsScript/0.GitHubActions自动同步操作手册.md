# GitHub Actions 自动同步操作手册

## 第一步：开启仓库写入权限

1. 进入 GitHub 仓库页面，点击右上角的 **Settings**。
2. 在左侧菜单栏，找到 **Actions → General**。
3. 滚动到页面底部，找到 **Workflow permissions** 部分。
4. 选择 **Read and write permissions**（读写权限）。
5. 点击 **Save**。

## 第二步：创建自动化工作流文件

1. 确保你在 `main` 分支。
2. 点击 **Add file → Create new file**。
3. 在文件名输入框中完整输入：`.github/workflows/sync-to-pages.yml`
4. 将以下代码粘贴到编辑器中：

```yaml
name: Sync to Public Pages

on:
  push:
    branches:
      - main
    paths:
      - 'file.md'  # 只有修改这个文件时才会触发同步

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout main branch
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Configure Git
        run: |
          git config --global user.name "github-actions[bot]"
          git config --global user.email "github-actions[bot]@users.noreply.github.com"

      - name: Sync file to gh-pages
        run: |
          git checkout gh-pages
          git checkout main -- file.md
          git add file.md
          git commit -m "chore: auto-sync file.md from main" || echo "No changes to commit"
          git push origin gh-pages
```

5. 点击 **Commit changes...** 提交文件。

## 第三步：检查同步结果

1. 修改 `main` 分支下的 `file.md` 并提交。
2. 点击仓库顶部的 **Actions** 选项卡。
3. 查看名为 **Sync to Public Pages** 的任务状态：
   - ✅ 绿色对勾：同步成功。
   - ❌ 红色叉号：同步失败，点击查看 `Error` 日志。

## 注意事项与常见问题

| 情况 | 处理方法 |
|------|----------|
| 文件名不同 | 将脚本中所有的 `file.md` 替换为实际文件名（如 `README.md`）。 |
| 多个文件 | 在 `paths` 下增加行，并在 `Sync file` 步骤中增加对应的 `git checkout main -- 文件名`。 |
| `gh-pages` 分支不存在 | 脚本会报错，请先手动创建一个名为 `gh-pages` 的空分支。 |
| 安全性 | 只要不把 `main` 分支的其他敏感文件写进脚本，它们永远不会出现在 `gh-pages` 分支。 |

## 最终访问链接

同步成功后，文件将通过以下格式公开访问：

```
https://<你的用户名>.github.io/<仓库名>/file.md
```
