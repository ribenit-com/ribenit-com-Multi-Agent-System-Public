# AI-Friendly 配置与函数库说明书

> 低学习成本方案

---

## 一、设计理念

| 原则 | 说明 |
|------|------|
| 双通道调用 | 人类/脚本使用英文变量名或函数名；AI 内部使用随机标志精准识别 |
| 随机标志用途 | 仅供 AI 内部识别，不显示在调用或输出中 |
| 低学习成本 | 人类只需记英文名，无需了解随机标志或内部映射 |
| 干净输出 | 调用结果只显示返回值或变量值，不输出随机标志 |

---

## 二、变量管理

### 定义方式

```bash
# 配置数组：随机标志|变量名|默认值
prompts=(
  "9f8b7c6a|DB_HOST|127.0.0.1"
  "2a4d1e3f|DB_PORT|5432"
  "5c7f8a1b|DB_USER|admin"
  "6d3e2b9c|DB_PASS|password"
  "1a9b3c7d|LOG_DIR|./logs"
  "8e2f5b4a|ENV|dev"
)
```

| 字段 | 说明 |
|------|------|
| `_id` | 随机标志，仅供 AI 内部识别 |
| `name` | 英文变量名，供人类/脚本直接调用 |
| `val` | 默认值 |

### 自动生成变量

```bash
for p in "${prompts[@]}"; do
    IFS="|" read -r _id name val <<< "$p"
    declare "$name=$val"
done
```

### 调用方式

```bash
# 人类调用
echo "$DB_HOST"

# AI 调用（可选）
echo "$(get_by_prompt 9f8b7c6a)"
echo "$(get_by_prompt DB_HOST)"  # 支持英文名直接调用
```

## 三、函数管理

### 定义函数

```bash
magic_hello() { local name="$1"; echo "👋 Hello, $name!"; }
magic_add()   { local a="$1" b="$2"; echo "$((a+b))"; }
```

### AI 内部映射随机标志

```bash
declare -A MAGIC_FUNCTIONS
MAGIC_FUNCTIONS["9a1b2c3d"]="magic_hello"
MAGIC_FUNCTIONS["4e5f6g7h"]="magic_add"
```

### AI 内部调用接口

```bash
call_by_prompt() {
    local id="$1"
    shift
    local func="${MAGIC_FUNCTIONS[$id]}"
    if [[ -z "$func" ]]; then
        echo "[WARN] 未找到随机标志: $id"
        return 1
    fi
    "$func" "$@"
}
```

## 四、使用示例

```bash
#!/bin/bash
source "./magic_env.sh"

# 变量调用（人类）
echo "数据库主机: $DB_HOST"
echo "数据库端口: $DB_PORT"
echo "数据库用户: $DB_USER"

# 函数调用（人类）
RES1=$(magic_hello "Alice")
RES2=$(magic_add 10 20)
echo "$RES1"  # 👋 Hello, Alice!
echo "$RES2"  # 30

# AI 内部调用（随机标志，可选）
RES3=$(call_by_prompt 9a1b2c3d "Bob")
RES4=$(call_by_prompt 4e5f6g7h 5 7)
echo "$RES3"  # 👋 Hello, Bob!
echo "$RES4"  # 12
```

## 五、特点总结

| 特点 | 说明 |
|------|------|
| 低学习成本 | 人类直接用英文名调用，无需关注随机标志或内部映射 |
| AI 精准识别 | 随机标志保证 AI 精准定位变量/函数，灵活内部使用无干扰 |
| 干净输出 | 只显示值或函数输出，不输出随机标志 |
| 易扩展 | 新增变量或函数只需在数组或映射中注册，调用方式不变 |

> 💡 随机标志只是 AI 内部的"咒语"，不影响人类使用。人类/脚本调用始终保持英文变量名或函数名，完全支持自动化脚本和 AI 智能调用。
