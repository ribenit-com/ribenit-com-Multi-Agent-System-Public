# 安装诊断检查清单

## 1️⃣ 安装前诊断（Pre-Installation Check）

**目标：** 确保环境准备好，避免安装/部署失败。

| 层级 | 检查内容 | 检查原因 |
|------|----------|----------|
| 集群基础层 | Kubernetes 版本、节点状态、网络连通性 | 集群必须健康、网络可访问 |
| 命名空间 | 目标 Namespace 是否存在 | Pod、Application 需要 Namespace |
| CRD / 资源支持 | ArgoCD CRD、Rollout CRD 是否已创建 | 控制器依赖 CRD，否则安装报错 |
| 存储类 / PVC 支持 | 是否有可用 StorageClass | PVC 创建失败会导致 Pod 启动失败 |
| 权限 / ServiceAccount | 是否有足够权限创建资源 | 安装过程可能需要 ClusterRole / RoleBinding |
| Git 仓库访问 | 仓库 URL 是否可访问，凭证是否正确 | ArgoCD 需要拉取 YAML 配置 |

> **核心：** 确保安装环境"能跑起来"，防止报错或失败。

---

## 2️⃣ 安装后诊断（Post-Installation Check / Health Check）

**目标：** 验证应用、控制器、Pod、服务是否按预期工作。

| 层级 | 检查内容 | 检查原因 |
|------|----------|----------|
| GitOps 层 | ArgoCD Application 同步状态、健康状态 | 确认应用已同步到集群、资源健康 |
| 控制器层 | Deployment / Rollout 状态、副本数、滚动更新、健康状态 | 确保控制器能管理 Pod，更新策略正常 |
| 存储层 | PVC 是否绑定、容量是否满足、Pod 可访问 | 确保应用存储可用 |
| Pod 层 | Pod 是否 Ready、容器状态正常 | 确保应用实例实际可用 |
| 应用层 | HTTP / API 可用性、端口开放、响应正确 | 确认应用对外服务正常 |

> **核心：** 确保安装部署完成后"能用起来"，对用户和服务可用性负责。
